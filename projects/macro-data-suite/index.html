What this tool does

Macro Data Suite is a streamlined interface for pulling a macroeconomic time series and running a consistent set of descriptive diagnostics (plots + summary tables) that help you sanity-check a series before modeling it. The live app is interactive (you choose a series, date window, transformations, and test settings); this page is a fixed snapshot.

Methodology (what you’re seeing)

The full project is built around a practical empirical workflow: (1) pull a time series, (2) transform it into a modeling-ready target (e.g., inflation from a price index), (3) run a small library of “recipes” (summary stats, simple regressions, forecasts), and (4) visualize what those outputs imply.

In this static demo, the exporter runs the project’s underlying pipeline on CPIAUCSL (CPI All Urban) over a multi-year window, then embeds the resulting Plotly figures and tables. See the export metadata above for the exact date range and commit.

Full app features (interactive version)

  • Provider workflows for market, macro, and news data pulls in one Streamlit app.
  • Series selection, date-range controls, merge options, and run-time configuration controls.
  • Recipe library for transformations, forecasting, and econometric outputs:
    • log_returns, rolling_zscore, sma, ema, rsi, macd, rolling_vol
    • inflation_yoy, inflation_mom_ann, growth_qoq_ann, spread, real_rate
    • ols, ar1, lp_irf
    • forecast_naive, forecast_drift, forecast_ets, forecast_arima, forecast_ridge_lags, forecast_rf_lags
    • news_sentiment (FinBERT daily sentiment pipeline)
  • Job-style execution with refresh/rerun behavior and artifact tracking.
  • News pull workflow includes query/language/max-record controls and optional sentiment scoring via FinBERT.

What this static page does not include

  • No provider switching or live data pulls from this page.
  • No job queue interaction (submit, refresh, rerun, or live status updates).
  • No parameter editing for recipes beyond the pre-exported scenario.

How to interpret (quick guide)

  • Levels vs. changes: macro series are often non-stationary in levels; growth rates / differences can be more stable.
  • Serial dependence: autocorrelation patterns inform whether AR/VAR-style dynamics are plausible.
  • Structural breaks: regime changes (policy, measurement, crises) can invalidate a single-parameter model fit.
  • Missingness/outliers: can create spurious spikes in logs/differences and distort inference.

Limitations of this static preview

  • No parameter controls: the series, transformations, and tests are frozen.
  • No new downloads: this site stores only the exported artifacts (and optional CSV snapshots).
  • Reproducibility depends on upstream data APIs and package versions; the repo documents requirements.

How to run the full interactive version

Clone the repo and run the Streamlit app locally:

git clone https://github.com/lucas-ms1/Macro-Project-1-Data-Suite.git
cd Macro-Project-1-Data-Suite

python -m venv .venv
# Windows:
.venv\\Scripts\\Activate.ps1
# macOS/Linux:
# source .venv/bin/activate

pip install -U pip
pip install -e ".[dev]"

cp .env.example .env
streamlit run streamlit_app.py

Repo: Macro Data Suite on GitHub

Note: macro downloads/forecasts use optional extras (e.g. .[macro], .[forecast]) as documented in the repo README.
